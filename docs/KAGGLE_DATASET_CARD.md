# CineLocalAI — Kaggle Dataset Card

This dataset card describes the curated Gold Dialogue Localization Dataset for CineLocalAI.

---

## 1. Dataset Description

The `cinelocalai_gold.jsonl` dataset is a highly cleaned and filtered corpus of **204 movie dialogue localization pairs** (English source dialogues and localized Hindi/Telugu equivalents). 

This dataset was generated by auditing the original 1,504-row Adaption Labs dataset, applying strict quality filters to discard non-dialogue domains, noise, and low-quality translations, and keeping only high-fidelity localized pairs with quality scores $\ge 4.0$.

---

## 2. Schema Explanation

Each record is stored as a JSON object with the following fields:

* **`source_text`** (String): The original English dialogue spoken in the film.
* **`target_lang`** (String): Target translation language (e.g. `hi` for Hindi, `te` for Telugu).
* **`source_lang`** (String): Source language (e.g. `en` for English).
* **`speaker`** (String): Identifier code for the speaking character.
* **`source_meta`** (Object): Segment metadata containing:
  - `segment_id`: Index of the segment.
  - `video_id`: Source video code.
  - `timestamp_start` / `timestamp_end`: Segment boundaries.
* **`row_searchable_text`** (String): Full instruction-tuning prompt structure including local terminology rules and translation targets.
* **Embeddings**: An array of 1536 float values representing the semantic vector space of the dialogue prompt.

## 4. Dataset Statistics

* **Total Records**: `204`
* **File Format**: `JSON Lines (.jsonl)`
* **File Size**: `5.95 MB`
* **Target Dialects**:
  - English to Hindi (`en_hi`): `102` records
  - English to Telugu (`en_te`): `102` records
* **Average Segment Length**: ~12 words

---

## 5. Intended Use Cases

1. **Fine-Tuning Translation LLMs**: Train models (like LLaMA or Mistral) on Indian cultural dialogue adapters.
2. **Localizing Subtitles**: Train text-to-text translators to match local slang, formality, and length constraints.
3. **Evaluating Movie Dubbing Quality**: Benchmark new pipelines against a validated human-graded localization ground truth.
